What this crop trains

p_user_has_floor is a dense state estimate over the complete user-owned turn, including continuation pauses. p_user_yield is valid only while that state is true and predicts whether the confirmed turn ends within 500 ms. Ambiguous or unsupported semantic regions are visibly masked instead of becoming false negatives.

At runtime, the assistant-floor phase reads p_user_has_floor to hand control to the user. Once the user owns the floor, the user-floor phase reads p_user_yield to release it; the floor-state output does not veto that transition.

Auxiliary supervision

Future activity is a soft occupancy fraction for four future horizons, not a hard speech/no-speech label. The interaction event is one categorical four-class auxiliary target placed on one causal anchor frame; its floor-take class is not a runtime state. Ambiguous boundaries, censored horizons, ineligible contexts, padding, and the four-second recurrent burn-in contribute no loss.
